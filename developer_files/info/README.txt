Конфигурация веб-сервера включает в себя несколько директив, которые определяют настройки сервера и его поведение при обработке запросов.

Директива "listen" определяет IP-адрес и порт для прослушивания. Можно указать порт, IP-адрес или имя домена, или их комбинацию. Например, "listen 123.23.43.1:8080;".
###
Директива "server_name" задает имена хоста, которые будут сравниваться со значениями в заголовке HTTP-запросов. Если IP-адрес не указан в "listen", то сервер также будет пытаться определить его здесь. Например, "server_name lol.com kek.org;".
###
Директива "client_max_body_size" задает максимальный размер тела запроса в байтах, мегабайтах и т.д. Например, "client_max_body_size 10m;".
###
Директива "location" определяет, какой блок конфигурации будет использоваться для обработки запросов, начинающихся с определенного URI. Например, "location /images { client_max_body_size 100m; }". Здесь при запросе файла, начинающегося с "/images", будет применяться другой максимальный размер тела запроса.
###
Директива "limit_except" определяет разрешенные HTTP-методы и используется только внутри "location". Например, "limit_except GET, POST;".
###
Директива "root" определяет корневую папку. Например, "root /var/www/example.com;".
###
Директива "index" задает файл, который будет возвращаться при запросе директории. Например, "index index.html;".
###
Директива "autoindex" определяет, будет ли сервер генерировать страницу со списком файлов в директории, если не указан "index". Например, "autoindex on;".
###
Директива "return" используется для редиректов и указывает код ответа и значения для заголовка Location. Например, "return 301 /new-page;".
###
Директива "error_page" используется для настройки обработки ошибок HTTP-запросов. Например, "error_page 404 /errors/404.html;".
Пример полноценной конфигурации веб-сервера может выглядеть так:

server {
    listen 80;
    server_name example.com;
    root /var/www/example.com;
    index index.html;
    client_max_body_size 10M;
    
    error_page 404 /404.html;
    error_page 500 502 503 504 /50x.html;

    location / {
        limit_except GET POST;
    }

    location /uploads {
        limit_except POST;
        root /var/www/uploads;
